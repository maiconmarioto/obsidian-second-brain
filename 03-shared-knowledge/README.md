# Shared Knowledge

## Purpose
- Store reusable engineering knowledge that is not owned by one project.

## Good candidates
- architecture patterns
- integration contracts
- engineering playbooks
- coding conventions
- release checklists
- recurring debugging patterns

## Structure
- `architecture/` for cross-project architecture and systems thinking
- `integrations/` for external systems and internal service interaction notes
- `practices/` for process and engineering workflow guidance
- `domains/` for business or technical domain concepts

## Rule
- Write this folder as if a new project might depend on it next week.
